# Project: Anti-Overspending Expense Tracker
# Role: Lead Mobile Developer (Flutter/Dart)

## Core Objective
You are building a cross-platform mobile application designed to actively prevent the user from overspending. This is not just a passive ledger; the app must intercept bad financial habits using AI categorization, strict daily/weekly limits, and visual warnings.

## Execution Rules

Analyze the existing system before executing any new tasks.

You must adapt to the current codebase and not totally change it.

Only change the necessary code. If a component or logic block does not need to change to fulfill the prompt, do not change it.

Prioritize lightweight, offline-first performance using SQLite.

Ensure the UI uses adaptive widgets (Material for Android, Cupertino for iOS) so the app feels natively rendered on both platforms.
